## How to use this binder

Each SOP follows a consistent structure so you can quickly find what you need:

- **Purpose** — what the SOP achieves
- **Scope** — who and what it applies to
- **Responsibility** — roles accountable for execution
- **Procedure** — step-by-step instructions with identifiers
- **Checklist** — supervisory observation tool for compliance scoring
- **Brand notes** — deployment guidance for property teams (not auditable)

### SOP code format

Every SOP has a unique code that tells you the department, function area, and sequence number:

| Code prefix | Department | Example |
|-------------|-----------|---------|
| `OPR` | Operations | OPR-SAF-001 (Safety), OPR-FAC-002 (Facility), OPR-EMO-001 (Emotional Intelligence) |
| `GDN` | Guidance | GDN-CCG-001 (Concierge), GDN-CUR-003 (Curated Program Design), GDN-EMO-001 (Emotional Intelligence) |
| `SVC` | Service | SVC-SPA-001 (Spa), SVC-LNG-005 (Longevity), SVC-LIF-002 (Life Optimization), SVC-MVT-001 (Movement) |

### Step identifiers

Within each SOP, steps use short format tags so you can reference them precisely:

- `[P#]` — Procedure step
- `[C#]` — Checklist item
- `[S#]` — Scope item
- `[R#]` — Responsibility item
